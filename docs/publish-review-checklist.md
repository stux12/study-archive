# 발행 검수 체크리스트 (진행 중 → Published)

글을 `Published`로 바꾸고 워크플로를 실행하면 실제 사이트에 나갑니다. 상태를 올리기 전에 아래를 확인하세요. 항목은 **사람이 직접 볼 것**과 **자동으로 걸러지는 것**으로 나뉩니다.

## 상태 흐름 요약

```
시작 전 / 진행 중  →  Published  →  완료
```

- **진행 중**에서 이 체크리스트를 통과한 글만 `Published`로 올립니다.
- 확정되어 더 손댈 일이 없으면 `완료`로 두면 **그대로 고정**됩니다.
- 사이트에서 내리려면 **행을 삭제**하거나 **초안 상태로 되돌리세요**. (`Archived` 같은 상태는 없습니다)
- **`완료` 글을 고쳤다면** 실행 시 **`force_resync`** 를 켜야 반영됩니다.

## 1. 사람이 직접 확인

- [ ] **사실 확인**: 개념·수치·명령어가 정확한가? AI 초안의 오류를 직접 검증했는가?
- [ ] **제목**: 목록에서 봤을 때 내용을 바로 알 수 있는가?
- [ ] **요약**: 한두 문장으로 채워졌는가? (비어 있으면 목록 카드가 허전함)
- [ ] **슬러그**: 의도한 URL인가? 다른 글과 겹치지 않는가?
- [ ] **분야·카테고리·태그**: 분류가 맞는가? 필터·통계에서 잘 잡히는가?
- [ ] **학습일**: 목록 정렬(최신순) 기준이므로 실제 날짜인가?
- [ ] **본문**: 코드 블록·인용·표가 깨지지 않는가? 민감정보(토큰/키/사내자료)가 없는가?
- [ ] **시각자료**: 핵심 개념에 다이어그램·그림·표가 **1개 이상** 있는가? ([콘텐츠·시각자료 정책](content-visual-policy.md))
- [ ] **정리하자면**: 내 언어로 채웠는가? (AI가 채우는 칸이 아님)
- [ ] **이미지**: 캡션/대체 텍스트가 있는가? 용량이 과하지 않은가?

> 버그·트러블슈팅 글은 위 항목 중 **사실 확인·제목·요약·분류·민감정보**만 보면 충분합니다.

## 2. 자동으로 걸러지는 것 (동기화 스크립트)

`scripts/sync-notion.mjs`가 배포 전에 아래를 자동 검사합니다. 문제가 있으면 **동기화가 중단되어 배포로 넘어가지 않습니다.**

- **중복 슬러그** → 두 글이 같은 파일로 저장돼 하나가 사라지는 사고를 막기 위해 중단합니다. (고정된 `완료` 글과 겹쳐도 중단)
- **사이트에 올릴 글 0건**(`Published`·`완료` 모두 없음) → 상태값 오타·필터 문제로 전체 글이 지워지는 것을 막기 위해 중단합니다.
  (정말로 전부 비우려면 `ALLOW_EMPTY_SYNC=1`)
- **요약 없음 / 본문 없음** → 배포는 막지 않지만 경고로 표시됩니다.

또한 **직접 작성한 글**(frontmatter에 `notionId`가 없는 파일)은 Notion에 없어도 **자동 삭제되지 않습니다.**

## 3. 배포 전 미리보기 (권장)

바로 배포하지 말고, 먼저 **어떤 글이 나갈지** 확인하세요.

- **GitHub Actions**: `Notion 검수본 배포` 워크플로 실행 시 `dry_run`을 **켜면** 배포 대상 목록만 로그에 출력되고 실제 배포·커밋은 되지 않습니다.
  (로그에 `재생성 N건 · 고정(완료) M건` 형태로 표시됩니다)
- **로컬**: `npm run sync:notion -- --dry-run` (환경변수 설정 필요, 파일을 쓰지 않음)

미리보기 목록에 이상이 없으면 `dry_run`을 끄고 다시 실행해 실제 배포합니다.

## 4. 워크플로 옵션

| 옵션 | 기본값 | 설명 |
| --- | --- | --- |
| `sync_notion` | true | Notion 글을 먼저 동기화 |
| `dry_run` | false | 미리보기만 (배포·커밋 안 함) |
| `force_resync` | false | **`완료` 글도 다시 가져와 갱신** |
