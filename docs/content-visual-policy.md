# 학습 아카이브 콘텐츠·시각자료 정책

## 왜 이 정책인가 — 이중 부호화(Dual Coding)

같은 내용을 **글 + 시각자료로 동시에** 담으면 기억·이해가 훨씬 오래간다. 이 아카이브의 목적은 "새롭게 알게 된 정보를 내 것으로 만드는 것"이므로, **개념은 최대한 시각화**하는 것을 기본 원칙으로 삼는다.

## 1. 글 구조

**초보자 눈높이**로 쓴다. 전문용어는 쉬운 설명·일상 비유를 곁들이고, 아주 세부적인 항목은 접기 토글로 접어 부담을 줄인다.

- **한 줄 요약** — 이 글을 한 문장으로
- **왜 중요한가** — 배경/동기
- **핵심 개념** — 여기에 **시각자료 1개 이상**
- **예제 / 코드**
- **함정과 방지책** — 흔히 빠지는 실수와 해결
- **정리하자면** — 글을 읽고 **내 언어·비유로 직접** 채우는 칸

> **`정리하자면`은 AI가 채우지 않는다.** AI가 나머지를 정리하면, 그걸 읽고 떠오른 생각·비유를 직접 적는다. (능동적 정교화)

**넣지 않는 것**: 복습 질문, 다음 학습, 참고 자료 섹션.

### 버그·트러블슈팅 글은 예외

위 틀을 쓰지 않고 **간단하게** 쓴다. 과한 시각자료도 지양(콜아웃 정도).

- **증상** → **해결**(명령/방법) → **짧은 설명**(왜 됐나) → *(선택)* 그래도 안 되면

## 2. 시각자료 정책 (핵심)

- **개념 하나당 시각자료(다이어그램·그림·표) 최소 1개**를 목표로 한다.
- 종류별 사용처:

| 시각자료 | 언제 쓰나 |
| --- | --- |
| **Mermaid 다이어그램** | 흐름·구조·순서·관계 (플로우차트·시퀀스 등) |
| **이미지/스크린샷** | 실제 화면, 손그림, 도식 캡처 |
| **표** | 비교·정리 (개념 A vs B 등) |
| **콜아웃 박스** | 핵심 / 주의 / 팁 강조 |
| **접기 토글** | 곁가지·심화 내용, 세부 해석 |
| **코드 블록** | 예제 (언어별 하이라이트, 복사 버튼 자동) |

- **접근성**: 이미지에는 캡션/대체 텍스트를 넣는다.
- **성능**: 스크린샷은 과도하게 크지 않게(용량 최적화).

## 3. 파이프라인 지원 현황

`sync-notion.mjs`가 Notion 블록을 Markdown/HTML로 변환한다.

| 기능 | 상태 | 작성법(Notion) |
| --- | --- | --- |
| 문단·제목·리스트·체크박스·인용·코드·구분선·북마크 | ✅ | 기본 블록 |
| 인라인 서식(굵게·기울임·취소선·코드·링크) | ✅ | 텍스트 서식 그대로 |
| 콜아웃(아이콘·색) | ✅ | Notion 콜아웃 블록 |
| 접기 토글 → `<details>` | ✅ | Notion 토글 블록 |
| 표(Notion table) | ✅ | Notion 표 블록 |
| 이미지/스크린샷 | ✅ | 이미지 붙여넣기 (동기화 시 **저장소로 다운로드**) |
| Mermaid 다이어그램 | ✅ | **코드 블록 언어를 `Mermaid`로** 설정 |
| 코드블록 복사 버튼 | ✅ | 자동 (사이트에서 처리, Mermaid 제외) |
| 수식(KaTeX) | ❌ 미지원 | — |

> ⚠️ **이미지 만료 주의**: Notion 이미지 URL은 시간이 지나면 만료된다(서명 URL). 그래서 동기화 시 이미지를 **저장소로 내려받아** 로컬 경로로 참조한다. Notion URL을 그대로 쓰지 않는다.

## 4. 품질·발행 규칙

- 발행 전 [검수 체크리스트](publish-review-checklist.md)에서 **시각자료 1개 이상 / 만료 URL 없음 / 민감정보 없음**을 확인한다.
- `dry_run` 미리보기로 배포 대상과 경고를 먼저 본다.

## 5. 분류 택소노미 (3단계)

```
분야(대분류)  →  카테고리(중분류)  →  태그(키워드)
```

- **분야**: 글마다 **하나**. 메인 목록 필터와 **분야별 기록 그래프**의 기준. (예: `백엔드`, `CS·기초`, `인프라·DevOps`, `프론트엔드`, `AI`, `트러블슈팅` … 필요하면 추가)
- **카테고리**: 글마다 **하나**. 분야 안의 세부 갈래 (예: 분야 `백엔드` → 카테고리 `Java`). 사이트에서 **분야를 고르면 그 분야의 카테고리만** 보이도록 자동으로 좁혀진다.
- **태그**: 여러 개 가능. **기존 태그 재사용을 우선**해 파편화를 막는다.
- 관련 글은 본문에서 서로 링크해 지식 그물망을 만든다.
