# Notion 학습 데이터베이스 템플릿

노션에서 **새 데이터베이스 → 표(Table)** 를 만들고 아래 속성을 추가하세요. 속성 이름은 동기화 스크립트의 기본값과 일치합니다. (이름을 바꾸려면 GitHub Repository Variables로 재정의)

| 속성 | 유형 | 예시 | 역할 |
| --- | --- | --- | --- |
| 제목 | Title | Docker는 내부에서 어떻게 움직이고 있을까? | 글 제목 |
| 상태 | Status | 시작 전 / 진행 중 / Published / 완료 | 사이트 노출 제어 (아래 참고) |
| 분야 | Select | 인프라·DevOps | **대분류**. 메인 목록 필터·통계 기준 |
| 카테고리 | Select | 컨테이너 | **중분류**. 분야 선택 시 자동으로 좁혀짐 |
| 태그 | Multi-select | Docker, Kubernetes | 세부 키워드 |
| 슬러그 | Text | docker-internals | URL. 비우면 제목에서 자동 생성 |
| 요약 | Text | 컨테이너는 격리된 프로세스다 | 글 목록 카드의 소개 문구 |
| 학습일 | Date | 2026-07-16 | 목록 정렬 기준(최신순) |

분류는 **분야 → 카테고리 → 태그** 3단계입니다. 자세한 기준은 [콘텐츠·시각자료 정책](content-visual-policy.md)을 참고하세요.

## 상태 흐름 (중요)

```
시작 전 / 진행 중  →  Published  →  완료
   (초안, 비공개)     (발행·매번 갱신)   (확정·고정)
```

| 상태 | 사이트 | 동작 |
| --- | --- | --- |
| `시작 전` · `진행 중` | ❌ 안 올라감 | 초안. 마음껏 고쳐도 됨 |
| `Published` | ✅ 올라감 | 실행할 때마다 Notion에서 **다시 생성**해 커밋 |
| `완료` | ✅ 남아있음 | 확정본. **재조회하지 않고 고정**(수정해도 반영 안 됨) |
| 행 삭제 · 초안으로 되돌림 | 🗑️ 제거 | 다음 실행 때 글·이미지 삭제 |

- 검수를 통과하면 `Published`로 올리고, 워크플로를 실행해 발행합니다.
- 더 손댈 일이 없으면 `완료`로 두면 그대로 고정됩니다. (`Archived` 같은 별도 상태는 없습니다 — 내리려면 **행 삭제**하거나 초안 상태로 되돌리세요)
- **`완료` 글을 고쳤다면** 워크플로 실행 시 **`force_resync`** 옵션을 켜야 반영됩니다.

> 발행 전 확인 절차는 [발행 검수 체크리스트](publish-review-checklist.md)를 참고하세요.

## 새 글 템플릿

Notion 데이터베이스의 새 템플릿 본문으로 넣어두면 일관된 기록을 만들 수 있습니다.

```md
## 한 줄 요약

## 왜 중요한가

## 핵심 개념   ← 시각자료 1개 이상

## 예제

## 함정과 방지책

## 정리하자면   ← 글을 다 쓴 뒤, 내 언어·비유로 직접 채우는 칸
```

- **`정리하자면`** 은 AI가 채우지 않습니다. 정리된 글을 읽고 **스스로의 비유·정리**를 적는 칸입니다.
- **버그·트러블슈팅 글**은 이 틀을 쓰지 않습니다. `증상 → 해결 → 짧은 설명 → (안 되면)` 정도로 **간단히** 씁니다.

## 시각자료 작성법 (Notion에서)

본문에 아래 블록을 그대로 쓰면 사이트에서 시각자료로 렌더됩니다. 자세한 정책은 [콘텐츠·시각자료 정책](content-visual-policy.md) 참고.

- **다이어그램**: `/code` 코드 블록을 만들고 **언어를 `Mermaid`로** 지정 → 플로우차트·시퀀스 등으로 렌더.
  ```
  flowchart LR
    A[정리] --> B{검수} -->|통과| C[Publish]
  ```
- **콜아웃**: `/callout` 블록. 아이콘(이모지)과 색이 유지됩니다(초록=팁, 빨강·주황=주의, 그 외=참고).
- **접기 토글**: `/toggle` 블록. 제목이 요약, 안쪽이 접히는 내용 → 곁가지·심화 설명에 사용.
- **표**: `/table` 블록. 첫 행을 헤더로 두면 표 머리로 인식됩니다.
- **이미지**: 붙여넣기/업로드. 동기화 때 **저장소로 내려받아** 저장하므로 링크가 깨지지 않습니다. 캡션을 달면 그림 설명으로 표시됩니다.
- **코드 블록**: 사이트에서 **복사 버튼이 자동으로** 붙습니다.
- **인라인 서식**: 굵게·기울임·취소선·인라인 코드·링크는 그대로 반영됩니다.

## 연결 권한 확인

Notion Developer Portal에서 Internal integration을 만든 뒤, 이 데이터베이스에 연결을 추가해 **Read content** 권한을 주세요. 연결 권한이 없으면 GitHub Action이 페이지를 찾지 못합니다.
